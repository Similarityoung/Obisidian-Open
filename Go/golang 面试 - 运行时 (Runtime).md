# golang 面试：运行时（Runtime）

[[Go/golang 面试]]

## 1. GMP：为什么要有 P

> [!note]
> 这一节先记一个最关键的结论：**P 的本地队列是 per-P 的，里面放的是 ready-to-run 的 goroutine，也就是 runnable 的 G。**

### 1.1 先记角色分工

- `G`：goroutine，任务本体
- `M`：线程，真正执行代码的 worker thread
- `P`：调度资源，负责承接运行 Go 代码所需的上下文，并带着一份本地 run queue

可以先把它记成：

```text
M 想跑 Go 代码，必须先绑定一个 P
P 手里有本地队列
本地队列里放的是待运行的 G
```

### 1.2 为什么 GMP 里要有 P

从历史上看，早期可以把调度简化理解成 `GM` 模型：

- `G` 是 goroutine
- `M` 是内核线程
- 所有待运行的 `G` 都更多依赖全局队列分发

这样的问题是：

- 全局队列容易形成热点
- 并发大时会带来更重的竞争
- 调度局部性也比较差

引入 `P` 之后，运行时多了一层“调度资源”：

- `M` 不再直接到处找 `G`
- `M` 需要先拿到 `P`
- `P` 自己带一份本地队列
- 调度状态因此可以分散到各个 `P` 上，而不是全堆在一个全局点上

这也是为什么很多文章会把 `P` 理解成“调度器上下文”或者“中间层”。

### 1.3 P 的本地队列里到底放的是什么

这点非常容易被问，而且很容易说混。

正确答案是：

> **本地队列里放的是可运行的 goroutine，也就是 ready-to-run / runnable 的 `G`。**

`runtime/proc.go` 一开头就把调度器的工作写得很明确：

- 调度器要把 **ready-to-run goroutines** 分发到 worker threads 上
- 调度状态是分布式的，尤其包括 **per-P work queues**

把这两句话合起来理解，结论就是：

> per-P 的本地工作队列里，装的就是“这个 P 手边待执行的 G”。

你可以直接记成：

```text
P 的本地队列 = 这个 P 接下来可以运行的 G 列表
```

例如有 4 个 `P` 时，可以粗略想象成：

```text
P0 -> [G1, G2, G3, ...]
P1 -> [G8, G9, ...]
P2 -> [G4, G5, ...]
P3 -> [G6, G7, ...]
```

然后某个 `M` 一旦绑定了某个 `P`，就会优先从这个 `P` 的本地队列里拿 `G` 来执行。

一句话背诵版：

> **本地队列是 per-P 的，里面放的是 ready-to-run 的 goroutine（G）。**

### 1.4 一个 G 是怎么进入某个 P 的本地队列的

结合你给的文章，可以先记这几个高频路径：

#### 新建 goroutine

新建 `G` 时，新的 goroutine 会优先加入当前 `P` 的本地队列。

#### 本地队列满了

如果本地队列满了，运行时会把本地队列中的一部分 `G` 挪到全局队列，以缓解单个 `P` 的局部拥塞。

#### 本地队列空了

如果某个 `P` 的本地队列已经空了：

- 会尝试从全局队列拿一部分 `G`
- 如果全局队列也不够，还可能去别的 `P` 那里做 work stealing

#### M 的执行顺序

`M` 绑定 `P` 之后，通常会：

1. 先看当前 `P` 的本地队列
2. 再看全局队列
3. 再考虑其他来源，例如偷别的 `P` 的工作

所以调度的核心倾向始终是：

> **优先消费本地 run queue，尽量减少全局竞争，并保持执行局部性。**

### 1.5 为什么这个设计更稳

把调度状态分散到 per-P 本地队列后，带来的收益主要有：

- 降低对全局队列的依赖
- 减少热点竞争
- 提高局部性
- 让空闲 `M/P` 更容易通过偷取机制获得工作

所以 `P` 的价值不只是“多了一个字母”，而是把调度从“更中心化的分发”变成了“带本地队列的分布式调度”。

### 1.6 面试速答版

- GMP 里 `G` 是 goroutine，`M` 是线程，`P` 是调度资源
- `M` 想执行 Go 代码，必须先绑定 `P`
- `P` 自带本地 run queue
- **P 的本地队列里放的是 runnable 的 G，不是 P，也不是 M**
- 调度器会优先从本地队列取 `G`
- 本地队列空了再看全局队列，或者从别的 `P` 偷工作
- `P` 的意义是把调度状态分散开，减少全局竞争并提升局部性
