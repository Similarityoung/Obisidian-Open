# 准备清单

## 你真正需要准备的 6 类东西

### 1. 流程管理

- 公司列表
- 当前阶段
- 截止时间和约面时间
- 待跟进事项

### 2. 个人材料

- 简历主版
- 针对不同方向的简历变体
- 1 分钟自我介绍
- 3 分钟项目讲解

### 3. 项目表达

- 每个项目的背景、目标、方案、权衡、结果
- 可量化结果
- 你个人贡献的边界

### 4. 技术准备

- 八股和基础
- 算法和手撕题
- 系统设计或场景题
- 项目追问

### 5. 公司研究

- 业务方向
- 团队情况
- 岗位关键词
- 面试官可能关注的点

### 6. 决策标准

- 城市
- 薪资
- 岗位成长
- 团队氛围
- 工作强度

> [!tip]
> 如果你现在时间有限，优先保证三件事：总览表、公司笔记、单场复盘。先把流程管住，再追求准备深度。

## 建议怎么做

1. 先建 3 个固定产物：公司笔记、单场复盘、总览表。
2. 每家公司都围绕 JD 做针对性准备，不做泛化努力。
3. 所有题目按答得好、卡住、完全不会三类整理。
4. 被连续问到 2 次以上的问题，沉淀成自己的标准答案。

## 面前一天检查

- [ ] 我能在 60 秒内讲清自己的经历吗
- [ ] 我能在 3 分钟内讲清最强项目吗
- [ ] 我知道这个岗位最看重什么吗
- [ ] 我准备了 3 个高质量反问吗
- [ ] 我知道自己这一轮最容易被追问的点吗

## 反问清单

- 这个岗位前 3 个月最期待我解决什么问题？
- 团队当前最核心的技术或业务挑战是什么？
- 对这个岗位的优秀表现，通常如何评价？
- 后续流程和时间安排是什么？
