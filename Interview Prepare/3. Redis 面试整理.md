#### 数据类型

![image.png](https://img.simi.host/20260402210439853.png)

| 数据结构            | 核心特点                           |      是否有序 |      是否可重复 | 典型场景                | 常见命令                                          | 备注               |
| --------------- | ------------------------------ | --------: | ---------: | ------------------- | --------------------------------------------- | ---------------- |
| **String**      | 一个 key 对应一个 value；可存字符串、数字、二进制 |         否 |        不涉及 | 缓存、计数器、分布式锁、Session | `SET` `GET` `INCR` `DECR`                     | 最常用，性能高          |
| **List**        | 双端链式/线性列表，支持头尾插入弹出             |         是 |          是 | 消息队列、任务队列、时间线、评论列表  | `LPUSH` `RPUSH` `LPOP` `RPOP` `LRANGE`        | 适合简单队列           |
| **Hash**        | field-value 映射，适合存对象属性         |         否 | field 不可重复 | 用户信息、商品信息、配置项       | `HSET` `HGET` `HGETALL` `HINCRBY`             | 很适合结构化数据         |
| **Set**         | 无序集合，元素唯一                      |         否 |          否 | 去重、标签、共同好友、点赞用户集合   | `SADD` `SMEMBERS` `SISMEMBER` `SREM`          | 支持交并差集运算         |
| **ZSet**        | 带 score 的有序集合，按分数排序            |         是 |     元素不可重复 | 排行榜、延迟队列、热度排序       | `ZADD` `ZRANGE` `ZREVRANGE` `ZINCRBY`         | 排序和范围查询很强        |
| **Bitmap**      | 用位来表示状态，本质上基于 String 的位操作      | 按 bit 位理解 |        不涉及 | 签到、在线状态、活跃标记        | `SETBIT` `GETBIT` `BITCOUNT`                  | 非常省内存，适合布尔状态统计   |
| **HyperLogLog** | 近似去重计数                         |         否 |        不涉及 | UV 统计、独立访客估算        | `PFADD` `PFCOUNT` `PFMERGE`                   | 节省内存，但结果非精确      |
| **Stream**      | 消息流结构，支持消费组和消息确认               |         是 |          是 | 消息队列、异步任务、日志流       | `XADD` `XRANGE` `XREAD` `XGROUP` `XREADGROUP` | 比 List 更适合复杂消息消费 |
